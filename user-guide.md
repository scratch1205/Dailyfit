# DailyFit 用户使用与数据备份指南

## 一、您的数据完全掌控在本地

您可以完全放心，**DailyFit 是一款纯本地运行的应用，所有您的个人数据都仅保存在您自己的设备上，绝不会上传至任何远程服务器或云端**。我们坚信用户对其个人数据拥有绝对的控制权，因此从设计之初就杜绝了任何形式的数据收集或网络传输。

具体来说：

- **衣物图片与搭配记录**：您上传的每一件衣物照片、创建的每一个穿搭组合，均通过浏览器提供的安全机制（如 IndexedDB 或 File System Access API）直接存储于您的手机或电脑中。
- **无账号系统，无需网络**：本应用不设用户账号，不依赖任何后端服务，即使在完全离线的状态下也能正常使用全部功能。
- **真正的数据主权**：只有您能访问这些数据。重启应用、更换网络环境甚至卸载重装，都不会导致数据被第三方获取。

这种本地化架构不仅保障了您的隐私安全，也让 DailyFit 能够快速响应、流畅运行，真正做到“您的衣橱，由您做主”。

## 二、数据备份与恢复操作指南

为防止设备丢失、损坏或误操作导致数据遗失，**强烈建议您定期对 DailyFit 数据进行备份**。本应用提供完整的本地数据导出与导入功能，确保您的穿搭记录永久可恢复。

### 备份您的数据

请按照以下步骤创建数据备份：

1. **进入设置菜单**：在应用内找到“设置”或“数据管理”选项。
2. **选择导出备份**：点击“导出数据”或“创建备份”按钮。
3. **等待处理完成**：系统将自动执行以下操作：
    - 收集所有衣物信息（`ClothingItem`）和搭配记录（`OutfitRecord`）的元数据
    - 打包所有关联的衣物图片文件
    - 生成一个包含 `data.json` 和 `images/` 目录的压缩文件
4. **保存备份文件**：下载生成的 `.zip` 文件，并将其**安全地存储在您的云盘、电脑或其他外部设备上**。

> **备份文件说明**：备份包为标准ZIP格式，内部包含 `data.json`（JSON格式的结构化数据）和 `images/` 文件夹（存放所有衣物图片）。此设计确保了跨平台兼容性与数据完整性。

### 恢复您的数据

当您需要在新设备上使用 DailyFit 或恢复历史数据时，请按此流程操作：

1. **准备备份文件**：确保您已将之前创建的 `.zip` 备份文件传输到当前设备。
2. **进入设置菜单**：打开 DailyFit 应用，进入“设置”区域。
3. **选择导入备份**：点击“导入数据”或“从备份恢复”按钮。
4. **选择并上传文件**：从设备中选择您的 `.zip` 备份文件。
5. **确认恢复**：系统将自动解压并验证文件内容，然后将所有衣物和搭配记录重新载入本地数据库。完成后会提示“数据恢复成功”。

### 备份频率建议

- **首次完整备份**：在添加超过10件衣物或创建首个搭配后立即执行。
- **常规更新备份**：建议**每两周或每月**执行一次常规备份。
- **重大变更前备份**：在计划更换手机、重置系统或进行大版本应用更新前，务必先创建一份最新备份。

通过以上操作，您可以完全掌控自己的数据生命周期，实现真正的“个人数字衣橱”自由迁移。

## 三、核心功能使用说明

DailyFit 应用专为简化您的穿搭管理流程而设计，三大核心功能模块——衣橱管理、搭配工作台与穿搭日历——协同工作，帮助您轻松记录、规划和回顾每日穿搭。

### 衣橱管理：构建您的数字衣橱

在这里，您可以上传并分类管理所有衣物单品，打造专属的数字化衣橱。

- **上传衣物**：
    - 点击“添加衣物”按钮，可从手机相册选择已有照片，或直接使用摄像头拍摄新衣物。
    - 上传后，系统将自动压缩图片至适合预览的尺寸（最大宽度800px），确保流畅加载。
- **分类管理**：
    - 所有衣物自动分为两大类：`Tops`（上衣）和 `Bottoms`（裤子/裙子）。
    - 在网格布局中浏览您的衣橱，点击任意单品可查看大图。
- **编辑标签**：
    - **长按**任意衣物卡片，可弹出操作菜单。
    - 选择“编辑标签”，可为衣物添加如“休闲”、“通勤”、“冬季”等自定义标签，便于后续筛选。
    - 支持随时修改或删除标签，让管理更灵活。

## 四、隐私与安全承诺

DailyFit 的设计核心是**尊重您的隐私权与数据主权**。我们不收集、不分析、更不会出售您的任何信息。以下是我们在隐私与安全方面做出的郑重承诺：

- **零数据收集政策**  
  应用运行过程中**不会发起任何网络请求**，所有功能均在本地完成。您上传的衣物照片、创建的搭配记录、添加的标签等全部数据，自始至终仅存在于您的设备之中。

- **端到端本地架构保障**  
  采用浏览器提供的 IndexedDB 和（若支持）File System Access API 进行数据存储，确保从图片上传、处理到展示的全流程都在您的设备内闭环完成，无任何数据外泄路径。

- **用户拥有完全控制权**  
  您可以随时通过设置菜单导出全部数据（JSON + ZIP格式），实现自由迁移；也可一键清除本地数据，彻底抹除所有痕迹。没有任何“隐藏备份”或“云端同步”，一切由您说了算。

- **无账号、无追踪、无广告**  
  无需注册登录，不使用 Cookie 或第三方分析工具（如 Google Analytics），杜绝任何形式的用户行为追踪。您就是唯一的使用者和管理者。

- **PWA 安全上下文运行**  
  作为一款渐进式网页应用（PWA），DailyFit 在独立的安全沙箱中运行，与浏览器其他标签页隔离，进一步降低潜在风险。其 manifest.json 配置确保了应用以 `standalone` 模式启动，提供类原生应用的安全体验。

我们相信，真正的个性化服务应建立在绝对信任的基础上。DailyFit 不仅是一款穿搭工具，更是您个人数字生活的可信赖伙伴。

## 五、常见问题与故障排除

为确保您能顺畅使用 DailyFit 应用，以下列出了一些常见问题及其解决方案。请按步骤操作，大多数问题均可快速解决。

### 数据丢失或未保存

- **问题现象**：添加的衣物或搭配记录在刷新后消失。
- **解决方案**：
    - 确保操作完成后等待系统提示“保存成功”，避免在加载过程中关闭页面。
    - 若已发生数据丢失，请立即尝试从最近的备份文件恢复（详见第二章）。
    - 检查浏览器是否启用了隐私模式（无痕浏览），该模式下部分本地存储可能无法持久化，请改用常规浏览模式使用应用。

### 图片无法加载或显示空白

- **问题现象**：衣橱中衣物图片显示为占位符或加载失败。
- **解决方案**：
    - 检查网络连接状态，尽管应用支持离线使用，首次加载资源仍需联网。
    - 清除浏览器缓存并重新加载页面，路径：设置 > 隐私与安全 > 清除浏览数据。
    - 确认设备存储空间充足，若空间不足可能导致图片写入失败。
    - 尝试重新上传该衣物图片，系统将自动处理压缩与存储流程。

### 应用无法离线使用

- **问题现象**：断网后无法打开应用或功能异常。
- **解决方案**：
    - 首次使用时请保持联网，并完整加载一次应用界面，以便 Service Worker 成功缓存核心资源。
    - 检查 PWA 安装状态，建议将应用“添加到主屏幕”以获得最佳离线体验。
    - 若仍失败，请前往浏览器设置中检查站点权限，确保未阻止本地存储和后台同步。

### 浏览器兼容性问题

- **问题现象**：在某些浏览器上功能异常或界面错乱。
- **解决方案**：
    - 推荐使用 **Chrome 或 Edge 浏览器** 以获得最完整功能支持，特别是 File System Access API 的高级存储能力。
    - 在 iOS Safari 上，部分手势操作可能受限，建议通过点击交互完成操作。
    - 避免使用老旧版本浏览器，请保持浏览器更新至最新版以支持现代 Web 标准。

### 存储空间不足警告

- **问题现象**：上传图片时提示“存储配额已满”。
- **解决方案**：
    - 浏览器对单个站点的存储空间有限制（通常为设备总空间的50%~80%），请检查设备剩余容量。
    - 删除不必要的衣物图片或旧搭配记录以释放空间。
    - 导出完整备份后，可选择清除本地数据并重新导入精简版内容。

### 应用更新后出现异常

- **问题现象**：更新后界面错乱、功能失效或数据不显示。
- **解决方案**：
    - 强制刷新页面（Ctrl+F5 或 Cmd+Shift+R），确保加载最新版本资源。
    - 检查是否有新的数据库结构变更，系统通常会自动迁移旧数据，但建议更新前先执行一次完整备份。
    - 如问题持续，请卸载并重新安装 PWA 应用，然后通过备份文件恢复数据。

## 六、移动端使用优化建议

为了获得最佳的 DailyFit 使用体验，特别是在手机等移动设备上，请参考以下优化建议。这些技巧将帮助您提升应用性能、节省存储空间，并充分利用其离线功能。

### 针对手机端的使用技巧

- **利用快捷方式快速操作**：  
  应用已配置 PWA 快捷入口，您可以通过主屏幕图标直接进入“添加新搭配”或“查看今日穿搭”，无需层层导航。
- **善用长按手势**：  
  在衣橱页面**长按**衣物卡片可快速调出编辑菜单，进行删除或标签修改，操作更高效。
- **分类浏览与标签筛选**：  
  切换 `Tops` 和 `Bottoms` 分类标签，结合自定义标签（如“夏季”、“工作日”）进行筛选，快速找到目标单品。

> 💡 **小贴士**：在通勤途中或购物时，随时打开应用记录灵感搭配，数据会自动保存至本地。

### 性能与存储优化建议

- **定期清理不常用衣物**：  
  若衣橱中衣物过多，可能影响加载速度。建议将季节性或已淘汰的衣物导出备份后暂时移除，需要时再恢复。
- **避免单次上传大量图片**：  
  建议每次上传控制在5-10张以内，系统需逐张压缩处理（最大宽度800px），分批上传可减少卡顿。
- **关闭非必要浏览器扩展**：  
  某些广告拦截或脚本管理插件可能干扰本地存储写入，若遇保存失败，尝试在无痕模式下运行应用。

### 离线使用最佳实践

- **首次使用请联网加载**：  
  虽然支持离线，但首次打开时需联网让 Service Worker 缓存核心资源，确保后续无网可用。
- **保持PWA安装状态**：  
  已“添加到主屏幕”的应用比普通网页标签页拥有更高的缓存优先级和更稳定的运行环境。

### 添加到主屏幕的方法

不同浏览器操作略有差异，请根据您的设备选择：

- **Android Chrome/Edge 用户**：
    1. 打开 DailyFit 网站
    2. 点击浏览器菜单中的“添加到主屏幕”选项
    3. 确认添加，即可在桌面看到独立应用图标

- **iOS Safari 用户**：
    1. 打开 DailyFit 网站
    2. 点击底部分享按钮
    3. 选择“添加到主屏幕”
    4. 自定义名称后点击“添加”

> 📱 **提示**：通过主屏幕图标启动的应用将以 `standalone` 模式运行，界面无浏览器边框，体验更接近原生App，且后台缓存更稳定。
